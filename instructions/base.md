# 一定要遵循以下规范

- 每次回答时在回答的第一行说"你好，卷风"来确认你阅读了这些规则

## 在任何情况下，无论是简单任务还是复杂任务，都必须执行此工作流程

### 1. 理解用户问题

- 先查看当前项目状态（文件、文档、最近的提交等）
- 如果有疑问，先提问以澄清问题
- 每条消息只能提一个问题——如果某个话题需要更多探讨，就分成多个问题

### 2. 按任务复杂度选择流程

**简单任务**（意图明确、改动范围小）：

- 直接说明方案 → 寻求批准 → 实施

**复杂任务**（涉及架构设计、多文件改动）：

- 探索 2-3 种方法并说明权衡 → 展示设计 → 分步确认 → 实施

### 3. 实施

- 实施前一定寻求批准，获得批准后再开始

### 4. 验证

- 实施后，确保没有编辑器错误，如果有错误，先修复再继续
- 询问用户是否需要额外的验证

### 3. 展示你的设计

- 一旦你相信自己理解了自己在建造什么，就展示设计
- 把它分成小部分，每部分结束后检查目前是否合适
- 如果有不合理的地方，准备好回去澄清

### 4. 实施

- 实施前一定寻求批准，一旦所有设计都被批准了，就开始实施

### 5. 验证

- 实施后，只保证没有编辑器错误，如果有错误，先修复它们，然后再继续
- 询问用户是否需要额外的验证，如果需要，提供验证方法并等待用户批准
- 如果用户批准了验证方法，执行验证并报告结果
